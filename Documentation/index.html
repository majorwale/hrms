Getting Started

Thank you for purchasing Workday. Workday a web based time clock app for accurate clock-in and clock-out. No more paper time cards. No expensive hardware. Hassle free attendance management.

If you have any question about Workday please feel free to contact with us. If you need support, customization or feature request, you may email to us at official.codefactor@gmail.com

System Requirements
  • PHP version 8.0.2 or higher
    • OpenSSL PHP Extension
    • PDO PHP Extension
    • Mbstring PHP Extension
    • Tokenizer PHP Extension
    • Ctype PHP Extension
    • JSON PHP Extension
  • MySQL 5.x or higher
  • Nginx or Apache (recommended http server)
Installation
  • Unzip the Download file from Codecanyon. You will find two folders, one folder contains Documentation & another is Application Files.
  • Upload the entire Application Files folder to your website or server.
  • Next you can rename the folder to whatever you like (HR, WORKDAY etc.)
  • Create a database user in your MySQL server and create a new blank database.

    Then import the database file from /Application/database/ folder named database.sql to your new empty database

    Grant the new user with all necessary permissions to the new database on your MySQL server.

    Use phpMyAdmin to simplify the operation.

  • Now visit the application folder. A sample .env.example config file is available here, rename the file to .env

    Open .env file with your text editor and fill the database name DB_DATABASE=mydatabase, username DB_USERNAME=myuser and password DB_PASSWORD=mypassword.

  • Next, please visit yourdomain.com/yourfolder or http://localhost/yourfolder to browse the web application.
  • Use demo accounts for first time login. Login with username: manager@example.com and password: demo12345. You may delete the demo manager and employee user account after creating a new manager type account.

Installation Guide for cPanel

  • Upload the entire Application files folder to your server using the File manager in your cPanel. To speed up the proccess compress the application files folder into application.zip, and then once uploaded in your server unzip the compressed file using the File manager.

  • Create a new database named "yourserver_workday" using cPanel MySQL Databases Administration.

  • Create a new user named "yourserver_workday_user" using cPanel MySQL Users Administration.

  • Give "yourserver_workday_user" database-user privileges to administer "yourserver_workday" database using cPanel Add User To Database feature.

  • Import "/application/database/database.sql" file to "yourserver_workday" database using phpMyAdmin.

  • Edit the "/application/.env.example" file. Rename this file to .env. Then, add database connection credentials.

  • Thats all. The installation process was simple. Browse your new application at https://yourdomain.com
  • Use demo accounts for first time login. Login with username: manager@example.com and password: demo12345. You may delete the demo manager and employee user account after creating a new manager type account.

Dashboard

Dashboard will provide you an overview of the application's key information.

  • Shows the total number of Regular and Trainee Employees.
  • Shows the total number of Online and Offline Employees.
  • Shows the total number of Approved and Pending Leave requests.

Company

  • To add a company, type Company name in the input box and click on SAVE button to add a Company.
  • Click on DELETE button to delete Company name.

Department

  • To add a department, type department name in the input box and click on SAVE button to add a Department.
  • Click on DELETE button to delete Department name.

Job Title

  • To add a job title, type job title name in the input box and click on SAVE button to add a Job Title.
  • Click on DELETE button to delete Job Title name.

Employees

  • Click VIEW button to view Employee profile
  • Click EDIT button to edit Employee information
  • Click on DELETE button to delete Employee profile and account.
  • Click ARCHIVE button to archive the Employee profile and account.
  • Click ADD button to add new Employee.
Employee Profile
  • Click on RETURN button to visit previous page.
Edit Employee Profile
  • Click on RETURN button to visit previous page.
  • Insert all related Employee information.
  • Click UPDATE button to submit updated Employee information.
  • Click on CANCEL button to cancel editing and return to previous page.
Add New Employee
  • Insert all related Employee personal information. Fill the required input fields highlighted in red input box.
  • Click SAVE button to submit Employee profile and information.
  • Click on CANCEL button to cancel adding and visit previous page.

Attendance

  • Click Manual Entry button this will open a new page where you can add new employee attendances.
  • Click Web Clock button this will open the web-based clock-in / clock-out page.
  • Click on DELETE button to delete Employee attendance information.
Web-based Clock In/Out

  • Click on the Clock-In or Clock-Out button. A notification will show after clocking in or out.

  • Click on CANCEL button to cancel, and return to previous page.
Edit Schedule

  • Click UPDATE button to update Employee Schedule.
  • Click on CANCEL button to cancel editing and return to previous page.

Reports

Settings

  • Select your time zone.
  • Choose your time format 12 hour or 24 hour.
  • Enable or disable RFID Clock in and out feature.
  • Enter your IP address to whitelist it from getting banned when you use IP restriction.
  • Click on SAVE button to update settings.

Troubleshooting

We are glad to help you if you have any questions relating to this application. We do our best to assist.

Blank pages

Blank pages can be caused by errors or if your server doesn’t meet the minimum requirements for Workday. To see exactly what the issue is, you need to see the error messages in the log file.

The Laravel log file is located at application/storage/logs/

Internal (500) Server Error

This might be caused by a simple problem between your database connection and PHP version.

You have to double check that your database connection is working, please check for miss-spelled database name, username, password, and host in the application/.env file.

500 server error also can be an issue with the PHP version you are using please review system requirements to check the right PHP version you need to install.

Images are not displayed and page has no styling

When images and front-end assets like css or js file is not loaded there can be problem in file permissions.

You have to make sure that correct file permissions are used in your server, folders must be set to 775, files must be set to 644 permissions, with exceptions to application/storage folder where this folder must be writable.
